Design Comparison Notes:

  • Kanban Style: Visual flow from left to right, shows bottleneck columns easily, great for production floor screens, supports drag-and-drop in future
  • Card List Style: More compact, easier to scan many work orders, familiar from current schedule page, better for detailed information display
  • Common Features: Both show current location, next location, priority/status indicators, and "Move to Next" button (disabled when at final step)
  • Complete Schedule Option: Checkbox allows user to choose whether to complete the schedule entry (checked = complete, unchecked = leave open)
  • Move Behavior: Button click → API call with complete_schedule: true/false based on checkbox → Card moves to next column or disappears if complete → Auto-refresh
  • User ID: Automatically pulled from logged-in user's session for audit trail in CETEC
